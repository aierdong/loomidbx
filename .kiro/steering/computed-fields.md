# 计算字段执行引擎（Steering）

**权威研究**：`docs/computed-field-research.md`（求值顺序、SQL 验证流程、Python 沙箱细节、错误文案与附录示例以该文档为准。）

**与表级引擎的关系**：`.kiro/steering/execution-engine.md` 描述 Planner 的**表间**依赖与拓扑；本文件描述单表内的**字段级**计算依赖与表达式执行策略。二者叠加：先生成普通列与外键相关列，再在第三阶段对 SQL/Python 计算列按字段 DAG 求值。

**updated_at**：2026-04-12 — 从 `docs/computed-field-research.md` 提炼为 steering；后续若实现与调研结论有偏差，先改详设再同步本文件结论行。

## 1. 产品边界

| 规则 | 说明 |
|------|------|
| MVP 范围 | MVP 阶段仅落地 SQL 表达式；Python 表达式延期到后续阶段（原因是阶段复杂度控制，而非方案不成立）。 |
| 引用范围 | 计算字段**仅可引用本表**其他列；禁止跨表引用（引导用户用外键等机制表达跨表语义）。 |
| 阶段划分 | 普通生成器 → 外键池采样 → **表达式求值**；同一阶段内 SQL 与 Python **不**分先后档，统一进入同一张字段 DAG。 |
| 环与自引用 | **禁止**保存与执行；配置保存时预检，生成前二次校验。 |
| 隐式依赖 | 除 `${...}` 外，若模板等配置间接引用列名，须单独建模或同样强制占位符，避免仅靠全文扫描列名。 |

## 2. 字段依赖与求值顺序

**核心模式**：`FieldDependencyGraph`（列 id → 节点、边、拓扑序）；Kahn 拓扑排序；可选按层分组并行（MVP 可串行）。

**引用解析（推荐且与实现挂钩）**：统一占位符 **`${column_name}`**，其中 `column_name` 匹配 `\w+`（**不支持**中文或特殊字符列名）。用正则提取依赖用于建边；**不对裸列名做全文扫描**，避免字符串字面量误匹配。

**求值替换**：批量生成/预览时将占位符换为实际值或字面量时，SQL 侧须**按类型转义或参数化**；文档中的字符串拼接仅为示意，实现以驱动参数化为准。

**列重命名**：占位符仍指向旧名时，校验阶段报「引用不存在」；UI 可提供批量替换（可与元数据迁移 spec 对齐）。

**错误信息风格**（表意即可，完整模板见详设）：简单环、多字段环、自引用、引用不存在字段等场景给出**可操作的**中文说明。

## 3. SQL 表达式：「解析」分层

| 层次 | 做什么 | MVP 不做什么 |
|------|--------|----------------|
| 依赖解析 | 仅从 `${column}` 抽边，服务 DAG/校验 | 不把多方言 AST 当作引用解析前提 |
| 合法性 / 方言 | **连目标库**执行 `SELECT <expr> AS result LIMIT 1`（或等价），配合**模拟值**代入 | 离线无连接时**禁止保存** SQL 表达式配置（把错误前移到设计期） |
| 静态约束 | 产品规则禁止子查询等；仅 `SELECT` 表达式语境 | 「无子查询」若需形式化证明再评估独立 SQL parser，与占位符方案正交 |

**语义边界**：`LIMIT 1` + 模拟值能挡掉大量语法/函数错误，但**不等于**全表语义等价（NULL、排序规则、时区、隐式转换等仍可能在真实行上与预览不一致），UI 应提示以目标库类型与数据为准。

**安全倾向**：危险函数/能力（如 `LOAD_FILE`、`INTO OUTFILE`）禁止；子查询为 MVP 硬约束，若未来放开需重新定义数据访问边界。

## 4. Python 表达式：沙箱策略

**运行时选型**：嵌入式方案以 **gpython** 为调研主线（无需外部 Python）；实现前必须用真实 API 核对 `ParseExpr` / `Eval` / AST 遍历等名称与能力。

**两层验证**：

1. **语法层**：仅接受**表达式**（Eval 语义），`import` / `def` / `class` 等在解析阶段拒绝。
2. **安全层**：遍历 AST，拦截黑名单**调用**与危险**属性**（如 `__class__`、`__globals__` 等）；`Call` 节点按解析出的函数名比对黑名单，而非禁止一切调用。

**能力模型**：预注入安全模块子集（如 `math`、`random`、`datetime`、`re`、`json` 等，以详设清单为准）；用户**不可**自由 `import`，仅能使用注入符号表；内置函数用白名单式收紧（危险名如 `open`、`eval`、`__import__` 等禁止）。

**随机数**：若预注入 `random`，种子应与**任务级/全局**生成器种子策略一致，避免与可复现性目标冲突。

**性能与隔离**：超时通过 goroutine + `select` 时，须认知子解释器可能**无法被取消**（泄漏与 CPU 风险）；MVP 若接受弱语义须在 spec 中写明。进程堆 `MemStats` **不等于**子解释器内存配额，不对用户承诺「硬沙箱内存上限」除非引入进程级隔离等强手段。

| 优先级 | 项 |
|--------|-----|
| P0 | 环检测（保存 + 执行前）、`${column}` 引用语法与解析、引用不存在字段的拦截 |
