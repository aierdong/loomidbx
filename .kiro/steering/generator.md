# 数据生成器（长期记忆）

## 权威文档

- **唯一权威详设**：`docs/generator.md`
- 本文件仅保留稳定决策与高频约束，避免与详设重复维护。
- 当本文件与 `docs/generator.md` 有冲突时，**以 `docs/generator.md` 为准**。

## 架构决策（必须遵守）

- 统一接口：`Generate(ctx)`、`GenerateBatch(ctx, count)`、`Reset()`、`Type()`。
- 主流程采用对象式生成器实例；`Preview(config, n)` 仅作为预览语法糖。
- 公共配置在生成器配置中显式声明，不从 Schema 隐式推断（扫描时可建议，但需用户确认）。
- Go 侧采用组合模式：具体生成器嵌入 `BaseGenerator` 复用公共处理。
- 扩展方式采用注册机制（编译时集成），不采用运行时插件动态加载。
- Go 与 Flutter 通过 FFI + JSON 通信，配置载体为 `GeneratorConfig`（按 `type` 分发解析）。

## 领域边界与语义约束

- 抽象类型边界：`int` / `string` / `decimal` / `datetime` / `boolean`。
- `CommonOptions.Unique=true` 语义是“目标表级唯一”，不是“本次生成唯一”。
- 表级唯一场景在任务开始前初始化 `UniquePool` 并加载已存在值后再生成。
- 发生唯一索引冲突（如并发竞态）时：任务失败，停止后续并执行回滚。
- 非字符串生成器输出到字符串列时必须通过 `Formatter` 明确格式。
- MVP 仅启用 SQL 计算表达式；Python 表达式延期是阶段复杂度控制，不是方案问题。
- AI/LLM 生成能力不纳入 MVP，仅保留扩展位。

## 类型体系（精简记忆）

- 整数：序列、范围随机、分布、雪花、位掩码
- 浮点：范围随机、分布
- 字符串：随机字符、正则、标识符、姓名/联系方式/证件/地理/网络/商业/文本
- 布尔：比例可配，支持 true/false 与映射输出（如 1/0、YES/NO）
- 时间：序列、范围随机、分布、相对时间、当前时间
- 通用：常量、带权重枚举、SQL 表达式（MVP）/Python 表达式（后续）、外键、自引用、外部数据、AI 生成（后续）

## 性能与质量基线

- 性能目标：>= 10,000 条/秒（口径：不包含外部数据源调用、不包含计算字段、包含唯一性约束检查、每批事务基准）。
- 必测维度：配置解析、批量性能、并发安全、FFI/JSON 链路、真实库约束（唯一/外键）。
- 可复现能力：MVP 支持全局 seed，并在运行历史中可追溯。

## 实施提醒

- 新增/调整生成器能力时：先更新 `docs/generator.md`，再决定是否同步本文件要点。
- 本文件面向“长期记忆”；避免写实现细节、阶段性任务拆解、临时方案。
