# Implementation Plan

## Tasks

- [x] 1. 元数据持久化与连接配置读写
- [x] 1.1 定义并实现连接记录在元数据存储中的创建、按标识更新与删除语义，保证连接标识在编辑过程中保持不变，并符合表前缀隔离约定。
  - 覆盖字段校验、事务边界以及持久化失败时的可观测错误返回。
  - _Requirements: 1.1, 1.2, 1.4_

- [x] 1.2 实现删除连接时的从属数据策略（删除连接采用”警示用户 -> 用户确认 -> 后端级联删除”固定策略），并在存在密钥环引用或加密列时执行相应清理或失效路径。
  - 用集成场景验证删除后不存在悬挂凭据引用。
  - _Requirements: 1.3, 3.2_

- [x] 1.2.1 实现密钥环写入流程（补充修复）：当用户选择 keyring 存储时，将凭据写入密钥环并仅在数据库中保存引用。
  - 新增 `KeyringAccessor.Set` 方法定义与实现
  - 新增 `credential_mode` 常量（aes/keyring/env_only）
  - 更新 `passwordForStorage` 支持写入 keyring 并返回更新的 extra JSON
  - 更新 `SaveConnection` 在更新时继承旧记录的 credential_ref
  - 单元测试覆盖：新建 keyring 存储、更新 keyring 凭据、keyring 不可用/拒绝访问
  - _Requirements: 3.2_

- [x] 2. 凭据解析与冲突策略 `(P)`
- [x] 2.1 实现环境变量占位与解析规则，固定「环境变量优先于存储值」等优先级表，并在变量缺失时返回可分类错误。
  - 提供可对表驱动的单元测试与不打敏感值的断言。
  - _Requirements: 4.1, 4.2_

- [x] 2.2 将解析优先级扩展为包含密钥环引用与列侧解密结果的完整决策链，覆盖两两冲突时的最终来源。
  - 在日志与错误路径中验证不落明文敏感值。
  - _Requirements: 3.1, 3.2, 4.3_

- [x] 2.3 实现密钥环不可用时的降级策略（提示、拒绝保存或回退到已支持模式），禁止无提示明文落库。
  - 各目标桌面平台至少覆盖一条可自动化执行的「拒绝访问」模拟路径。
  - _Requirements: 3.3_

- [x] 2.4 实现密钥环最小支持矩阵（补充任务）
  - 新增 `backend/app/keyring.go` 定义 `PlatformKeyringAccessor` 与 `KeyringPurger`
  - 新增 `backend/app/keyring_impl.go` 实现 Windows/macOS/Linux 平台适配（使用 `zalando/go-keyring`）
  - 新增 `backend/app/keyring_noop.go` 为非支持平台提供默认拒绝实现
  - 新增 `backend/app/keyring_test.go` 单元测试覆盖平台检测、错误映射、引用构建等
  - 平台后端映射：Windows (Credential Manager/DPAPI), macOS (Keychain Services), Linux (Secret Service/libsecret)
  - _Requirements: 3.1_

- [x] 3. 目标库连接器与连接测试 `(P)`
- [x] 3.1 按支持的数据库类型封装打开连接与 Ping 或等价可达性检测，并设置可配置短超时，避免长阻塞。
  - 成功与失败路径均返回统一结构化错误，不进行误报成功。
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 3.2 将凭据解析结果注入连接器参数拼装流程，对 TLS 与协议级错误进行归类映射。
  - _Requirements: 2.2, 4.2_

- [x] 4. 对外服务编排与 FFI JSON 契约
- [x] 4.1 实现聚合服务层，对「测试连接、列出连接、保存连接、删除连接」提供单一编排入口，并仅暴露 JSON 友好的载荷与错误外壳。
  - 错误对象需带稳定错误码以供后续统一 FFI 文档消费。
  - _Requirements: 1.1, 2.1, 2.2, 5.2_

- [x] 4.2 将内部类型适配为与权威文档一致的 JSON 请求与响应形状，保存与列表响应中永不包含明文密码或令牌。
  - _Requirements: 1.1, 4.3, 5.2_

- [x] 5. 自动化测试与质量基线 `(P)`
- [x] 5.1 为凭据解析优先级、环境变量缺失与脱敏日志撰写单元测试，覆盖主要组合矩阵。
  - _Requirements: 4.1, 4.2, 4.3_

- [x] 5.2 以嵌入式元数据存储配合容器或本地目标库样例做连接测试集成测试，覆盖成功与典型失败（认证、超时）。
  - _Requirements: 2.1, 2.2, 2.3, 1.4_

- [x] 5.3 建立 JSON FFI golden 或契约快照测试，冻结 `ok`/`data`/`error` 形状供后续契约演进回归。
  - _Requirements: 5.2_

- [x] 6. 跨 Spec 联调与边界假验
- [x] 6.1 与 **spec-02** 衔接假验：在仅导入连接子系统模块的前提下，提供「已获得合法数据库连接句柄」的测试桩，验证本阶段不触发任何 Schema 扫描或快照写入调用。
  - 产出联调检查清单项，供后续扫描实现对照。
  - _Requirements: 5.1, 5.3_

- [x] 6.2 与 **spec-06** 衔接假验：将本服务错误码映射示例与样例载荷固定为契约夹具，供 FFI 双向对齐评审。
  - _Requirements: 5.2, 5.3_

- [x] 6.3 与 **spec-07** 衔接说明：列出 UI 可调用的异步入口与阻塞时间预期，不要求在本阶段实现界面，但需提供接口级对接说明供前端排期。
  - _Requirements: 2.3, 5.3_
