## 一、基本方案（草案框架）

本节给出「若不另作说明时的默认叙事」，便于讨论时有一个共同起点；**不代表已拍板**。

### 1.1 数据源分类（四象限）

| 类型 | 典型形态 | 是否出网 | 典型凭据 |
|------|----------|----------|----------|
| **A. 本地/内嵌** | 内置词表、用户选择的本地 JSON/CSV | 否 | 无 |
| **B. 用户文件** | 用户磁盘上的 JSON/CSV 路径 | 否（读本地） | 无 |
| **C. HTTP(S) 可复现资源** | 固定 URL 的公开数据集、自建对象存储 | 是 | 可选（匿名或 Token） |
| **D. 交互式 API** | REST 维度表、LLM 补全 | 是 | API Key / OAuth 等 |

**基本方案**：四类在配置模型上统一为「**外部供给 ExternalFeed**」抽象，差异体现在 `kind`（`file` / `http_static` / `rest` / `llm`）、`auth`、`cache_policy`、`failure_policy` 子结构；生成器只消费「已解析的供给句柄」，不各自实现一套 HTTP 客户端逻辑。

### 1.2 生命周期（与执行引擎对齐）

| 阶段 | 行为（草案） |
|------|----------------|
| **配置保存** | 校验 URL 格式、文件存在性（可选：弱校验仅记录路径）、脱敏展示密钥；敏感字段加密落盘（与连接密码策略对齐）。 |
| **预览（小样本）** | 允许同步拉取；超时较短；可跳过缓存以反映「当前远端状态」。 |
| **批量生成** | 优先走缓存/批拉策略；受全局 `context` 超时与每源超时约束；失败策略按字段或按任务配置。 |

### 1.3 默认产品边界（草案）

- **离线优先**：用户未配置任何出网源时，应用行为与「无网络」一致，不隐式联网。
- **显式出网**：任何 `http(s)://` 或 LLM 调用须在 UI 中有可感知开关或能力声明（避免企业环境合规争议）。
- **密钥不入日志**：运行历史、调试日志中默认不记录完整 API Key、Authorization 头、请求体中的密码字段。

## 二、分场景的基本方案摘要

### 2.1 REST 维度表

- **用途**：按大纲从 CRM 等系统取「仓库名称」等维度值。
- **草案**：支持 GET（MVP）与可选 POST；支持 Header/Query 占位符中的「静态密钥」；响应体支持 JSONPath 或简化「数组字段」提取。
- **与性能关系**：默认不采用「每行一次 HTTP」；采用 **任务级或批次级拉取 + 内存池随机/轮询**（具体参数待决断 §5）。

### 2.2 本地 JSON / 用户文件

- **用途**：大纲中的 JSON 文件供给枚举或维度。
- **草案**：路径可为绝对路径或应用沙箱内相对路径；保存配置时可选「校验文件存在」；生成时文件缺失 → 归入失败策略（§4）。

### 2.3 公开 URL 数据集（CSV 等）

- **用途**：食品名称等「真实世界」词表。
- **草案**：与 `http_static` 同类；首次下载后可落本地缓存目录（路径与清理策略待决断）。

### 2.4 AI 生成（LLM）

- **用途**：如电影名等开放文本。
- **草案**：单独 `llm` kind，与 REST 区分（计费、限流、提示词模板不同）；**默认不参与**「万级/秒」热路径，或仅预览/小批量（与性能目标关系待决断 §4）。

## 三、横切能力：问题域与草案条目

### 3.1 网络访问

| 条目 | 说明 |
|------|------|
| 代理 | 是否支持 HTTP(S)_PROXY / 系统代理 / 应用内配置。 |
| TLS | 自签名证书是否允许「用户显式信任」导入。 |
| IPv6 / 内网 | 仅 RFC1918 地址是否允许（部分企业仅内网 CRM）。 |

### 3.2 认证与密钥存储

| 条目 | 说明 |
|------|------|
| 存储位置 | 与 `connections` 密码一致：元数据 DB + 加密；或支持仅环境变量注入不落库。 |
| 刷新 | OAuth refresh_token 是否纳入 MVP。 |
| 展示 | UI 仅显示掩码；导出配置是否允许包含密钥（应默认禁止或加密包）。 |

### 3.3 缓存

| 条目 | 说明 |
|------|------|
| 粒度 | 按 URL+Query+Header 规范化键 vs 按「供给 ID」单键。 |
| 位置 | 内存 / 用户缓存目录 SQLite / 裸文件。 |
| 失效 | TTL、手动刷新、配置变更即失效。 |

### 3.4 失败降级（Degrade）

| 条目 | 说明 |
|------|------|
| 超时 | 全局、每源、预览 vs 批量是否不同。 |
| HTTP 状态 | 4xx/5xx/429 是否区分重试与熔断。 |
| 业务空结果 | 空数组是错误还是触发降级占位符。 |
| 与约束联动 | 字段 `NOT NULL` 且无默认值时，降级为 NULL 是否一律禁止。 |

### 3.5 隐私、合规与日志

| 条目 | 说明 |
|------|------|
| 生成历史 | 是否记录「曾调用的 URL / 模型名」；是否记录 prompt 摘要哈希。 |
| LLM 侧 | 是否上传表结构、样例行、列注释；用户一键「仅发送模板不含业务数据」。 |
| 审计 | 企业版是否需要「谁、何时、调用了哪个外部端点」——与 MVP 范围划分。 |

## 四、可能的问题清单（讨论用）

下列问题**不要求在本文件内回答**；回答后可将结论写入 steering 或 `execution-engine.md` / `generator.md`。

### 4.1 产品与体验

1. MVP 是否包含 **LLM**？若包含，是否仅限「预览」与「单行实验」？
2. 用户是否必须能标记项目为 **「禁止出网」**（全局开关），且配置校验阶段即拦截？
3. 外部源失败时，默认是 **整任务失败**、**跳过该表**、还是 **该列占位符**（三选一或组合）？

### 4.2 技术与性能

4. REST 维度：「全量拉取一次」的上限（响应体大小、行数）超过时如何处理（流式、分页协议、拒绝配置）？
5. 同一任务内多表共享同一 REST 端点：是否 **合并为一次请求**（需 URL 完全一致）？
6. AI 与 **≥1 万条/秒** 目标冲突时，产品文案是否明确「AI 生成器不参与该性能承诺」？

### 4.3 安全

7. 是否允许用户在 URL 中嵌入 **查询串密钥**（易泄露到日志）？是否 UI 警告并推荐使用 Header？
8. 下载的公开数据集文件：是否需要 **校验哈希**（防投毒）或仅 HTTPS 信任链？

### 4.4 运维与跨平台

9. 缓存目录在 Windows/macOS/Linux 的 **默认路径** 与卸载时清理策略。
10. 企业代理需 **NTLM/Kerberos** 时是否在 MVP 明确「不支持」？

## 五、可选方案对比（摘要表）

### 5.1 REST 拉取策略

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **R1 每行请求** | 每生成一行调一次 HTTP | 实时性最强 | 极易触发限流与性能崩溃 |
| **R2 任务级全量** | 任务开始时拉一次，内存池抽样 | 简单、吞吐高 | 大响应内存风险；数据「任务内不变」 |
| **R3 分页/游标** | 协议化分页直至上限或耗尽 | 可控内存 | 需约定分页格式或适配器 |
| **R4 显式缓存 TTL** | R2 + 磁盘缓存跨任务复用 | 减少重复流量 | 一致性弱；需失效策略 |

### 5.2 失败时行为

| 方案 | 描述 | 适用 |
|------|------|------|
| **F1 硬失败** | 立即中止任务，已写入批次按事务回滚策略处理 | 强一致、测试环境 |
| **F2 列级 NULL** | 该列填 NULL（违反 NOT NULL 则失败） | 可空维度 |
| **F3 列级占位** | 填固定占位字符串如 `[UNAVAILABLE]` | Demo；可能违反 CHECK |
| **F4 降级内置词表** | 回退到内置列表 | 需维护映射关系 |

### 5.3 LLM 密钥来源

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **K1 仅加密落盘** | 与 DB 连接密码一致 | 体验统一 | 备份文件需同样保护 |
| **K2 仅环境变量** | 进程内读取，永不写入元数据 DB | 密钥不进应用 DB | 桌面端分发体验差 |
| **K3 系统密钥环** | Keychain/Credential Manager | 安全最佳 | 实现成本高 |

### 5.4 日志中外部调用记录

| 方案 | 描述 |
|------|------|
| **L1 最小** | 仅记录 supply_id、成功/失败、耗时 |
| **L2 中等** | + HTTP 状态、URL host（不含 query） |
| **L3 详细** | + 请求/响应摘要（需脱敏规则） |
