## 一、外部数据源的意义与特征

外部数据源用于拉取外部“维度数据”，典型场景包括：

- 从 CRM/HR 拉取销售员、组织、物流方式等主数据。
- 从公开数据集拉取药品名、城市名等标准词表。
- 从合作伙伴系统拉取可选项列表。

其共同特征为：

1. 数据量不大，通常在数十到数万条。
2. 属于关键信息，不接受降级补位。
3. 系统无法感知外部源是否发生增量变更。
4. 数据用于填充单个字段，通常仅取一列值，其它列在流程内被丢弃。

## 二、数据源分类（5 类）

> 统一抽象建议：`ExternalFeed`。  
> 与原草案不同，本节采用 **5 类**，不是 4 类。


| 类别                  | 描述                      | 是否出网  | MVP      |
| ------------------- | ----------------------- | ----- | -------- |
| 1. 内嵌 JSON/CSV 文件   | 应用内置静态文件（如全球城市）         | 否     | 是        |
| 2. 用户上传 JSON/CSV 文件 | 用户导入文件，独立于内嵌目录          | 否     | 是        |
| 3. HTTP(S) API/资源   | 企业内部数据集 + 公开无认证 URL 数据集 | 是     | 是        |
| 4. SQL 外部数据库        | 通过连接凭证 + SQL 语句取数       | 取决于网络 | 是        |
| 5. LLM 补全           | 任意 OpenAI 协议兼容模型        | 是     | 否（MVP 外） |


### 2.1 HTTP 认证方式（能力边界）

支持的认证类型：

- 无认证
- API Key（Header / Query）
- Bearer Token / OAuth 2.0（支持通过自定义 URL + POST 内容动态取 token）
- Basic Auth
- HMAC 签名
- Digest Auth

说明：

- OAuth 刷新机制不纳入当前范围；仅支持“按需重新授权”。
- URL 中嵌入 query 密钥允许，但应发出风险告警。

### 2.2 SQL 认证方式（能力边界）

支持的连接认证类型：

- 用户名/密码（最常见）
- 连接串内嵌凭据（DSN）
- 环境变量注入凭据（推荐生产）

说明：

- SQL 外部源以“连接凭证 + SQL 语句”作为最小执行单元。
- 不纳入 NTLM/Kerberos 等企业域认证的专项适配。
- 凭据支持直接输入并落库，或以环境变量名方式落库。

### 2.3 HTTP/SQL 返回数据格式要求


| 数据源类型    | 格式要求      | 单列提取规则                            |
| -------- | --------- | --------------------------------- |
| HTTP API | JSON 数组   | 配置 `field_path` 提取单个字段            |
| SQL      | 查询返回单列或多列 | 配置 `target_column`                |
| JSON 文件  | JSON 数组   | 配置 `field_path`                   |
| CSV 文件   | UTF-8 编码  | 配置 `column_index` 或 `column_name` |


补充规则：

- 所有类型最终都归一为“单列值数组”进入生成流程。
- `field_path` 解析后若字段不存在、值为 `null` 或结果为空数组，按失败语义处理。
- SQL 多列结果仅保留 `target_column` 指定列；未命中列名视为失败。
- CSV 同时配置 `column_index` 与 `column_name` 时，优先使用 `column_name`。

### 2.4 LLM 参数来源（非 MVP）

LLM 连接参数（`api_key`、`model_name`、`base_url` 等）支持：

- 直接输入并落库；
- 或通过环境变量注入（落库保存环境变量名）。

## 三、产品边界与横切能力（已确认）

### 3.1 产品边界

- **不要求离线能力**（无需离线优先叙事）。
- 外部源失败即整任务失败，不做降级填充。
- 外部数据在任务开始前一次性获取，不做跨任务缓存。

### 3.2 网络访问

- 不对代理、IPv6、内网、TLS 信任做额外专项支持。
- 网络行为依赖用户当前运行环境能力。

### 3.3 认证与密钥存储

- 支持环境变量注入。
- 也支持直接输入并落库（非生产环境优先考虑便捷性）。
- 不处理 OAuth refresh token 生命周期；仅支持动态授权获取 token。

### 3.4 日志与安全

- 日志记录 HTTP 状态码。
- URL 仅记录不含 query 的部分。
- 允许 URL query 携带密钥，但需提示风险。
- 公开数据集可选配置校验哈希；不强制依赖 HTTPS 信任链（考虑企业内网运行场景）。

## 四、统一执行语义（已确认）

### 4.1 拉取策略

- 任务开始时一次全量拉取。
- 默认行数上限：`10000`（可由环境变量调整）。
- 超过上限时截断并丢弃超限行，在配置/运行阶段发出告警。
- 不做请求合并策略。

### 4.2 字段提取策略

- 外部源仅用于目标字段的一列值填充。
- 非目标列在生成流程中直接丢弃。

### 4.3 失败定义与处理

以下任一情况视为失败：

1. 超时、连接失败、文件路径失效等。
2. HTTP 状态码不是 `200`。
3. 返回结果为空数组。
4. 返回结果中包含 `null` 值。

失败处理规则：

- 立即终止当前处理进程。
- 返回失败提示。
- 回滚事务。
- 失败语义与字段约束无关（即不走列级降级/占位策略）。

## 五、MVP 范围与阶段规划

### 5.1 MVP（当前版本）

- 包含：文件（内嵌/上传）、REST/HTTP、SQL。
- 不包含：LLM 补全能力。
- AI 生成器不纳入当前性能承诺范围。
- 出网策略：默认允许出网，无全局开关；执行生成任务时必须弹出确认框并提示“将访问外网”。
- 失败策略：硬失败（F1）。
- 缓存策略：无落盘缓存。
- 日志级别：L2（状态码 + 脱敏 URL）。

### 5.2 MVP 后扩展方向

- 增加 OpenAI 协议兼容 LLM 供给类型。
- 提供系统草案提示词与要点提示，最终提示词由用户确认。
- OAuth 复杂刷新策略可在后续版本评估。

## 六、可选方案对比（按当前结论收敛）

### 6.1 REST 拉取策略

- **选定**：任务级全量拉取一次（内存内处理），带行数上限。  
- **不选**：每行请求、跨任务缓存。

### 6.2 失败行为

- **选定**：F1 硬失败。  
- **不选**：列级 NULL、占位符、回退内置词表。

### 6.3 密钥来源

- **选定**：可落盘，也可环境变量加载（落盘保存变量名）。

### 6.4 日志

- **选定**：L2；包含 HTTP 状态，不含 query 的 URL。
